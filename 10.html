AI产品需求分析方法论

第十讲：模型需求与评估指标

模型类型选择、核心评估指标定义、模型迭代与监控需求

一、模型类型选择

选择合适的机器学习模型类型是AI产品成功的基石。主要分为三大类：

监督学习 (Supervised Learning)

适用场景：拥有大量带标签的数据，且输入与输出之间存在明确的映射关系。

  • 典型任务：分类、回归
  • 产品示例：垃圾邮件过滤、房价预测、图像识别
  • 关键需求：高质量标注数据、明确的预测目标

无监督学习 (Unsupervised Learning)

适用场景：数据无标签，需要发现内在结构、模式或分组。

  • 典型任务：聚类、降维、关联分析
  • 产品示例：客户细分、异常检测、推荐系统（部分）
  • 关键需求：数据的内在相似性度量、对“发现”结果的解释性

强化学习 (Reinforcement Learning)

适用场景：智能体与环境动态交互，通过试错学习最优策略以最大化累积奖励。

  • 典型任务：序列决策、游戏、机器人控制
  • 产品示例：自动驾驶、游戏AI、资源调度
  • 关键需求：定义明确的环境、奖励函数、可模拟或安全试错的场景
选择原则： 始终从业务问题出发，结合数据现状（有无标签、数据量、质量）、对可解释性的要求、计算资源约束以及迭代速度要求进行综合决策。

二、核心评估指标定义：业务指标与模型指标对齐

AI产品的价值最终体现在业务提升上，因此必须将模型的技术指标与核心业务指标（North Star Metric）紧密对齐。

2.1 常见模型技术指标

任务类型 核心模型指标 定义与说明
分类任务 准确率、精确率、召回率、F1-Score、AUC-ROC 衡量模型区分不同类别的能力。需根据业务代价选择侧重（如欺诈检测重召回，推荐排序重精确率）。
回归任务 均方误差(MSE)、平均绝对误差(MAE)、R²分数 衡量预测值与真实值之间的误差。MSE对异常值敏感，MAE更稳健。
聚类任务 轮廓系数、Calinski-Harabasz指数、戴维森堡丁指数 评估簇内紧密度和簇间分离度，无绝对“正确”标准，需结合业务解读。
推荐/排序任务 NDCG、MAP、点击率(CTR)、转化率 衡量列表排序质量及用户交互效果，直接关联用户体验与商业价值。

2.2 业务指标与模型指标对齐框架

对齐步骤：

  1. 拆解业务目标： 例如，业务目标是“提升电商平台营收”，可拆解为“提升客单价”和“提升购买频次”。
  2. 映射到用户行为与模型作用点： “提升客单价”可能依赖于“推荐系统提供高价值商品组合”，模型的作用是优化商品排序。
  3. 选择可量化的模型指标： 为“推荐排序”任务选择NDCG（衡量排序质量）和推荐商品的预估GMV（直接关联营收）。
  4. 建立监控与反馈闭环： 线上A/B测试，同时监控模型指标（NDCG）和业务指标（GMV变化），验证对齐关系。
注意： 模型指标的提升并不总是导致业务指标提升。需通过严谨的因果推断或A/B实验确认其因果关系，避免“指标游戏”。

三、模型迭代与监控需求

模型上线并非终点，而是持续迭代的起点。必须建立系统的监控和迭代机制。

3.1 模型监控核心维度

性能监控

  • 预测质量： 关键指标（如准确率、AUC）的时序变化。
  • 预测延迟与吞吐量： 响应时间、QPS是否符合SLA。
  • 资源消耗： CPU/GPU、内存使用率。

数据监控

  • 输入数据分布： 特征值范围、缺失率、与训练集分布的差异（数据漂移）。
  • 预测结果分布： 各类别预测概率的分布变化（概念漂移）。
  • 数据管道健康度： 数据更新是否准时、完整。

业务影响监控

  • 核心业务指标： 与模型相关的业务指标（如CTR、转化率）波动。
  • 异常告警： 设定阈值，对性能骤降或数据异常进行告警。

3.2 模型迭代流程

3.3 需求文档中需明确的要点

  • 监控指标清单： 明确需要实时监控和定期报告的技术与业务指标。
  • 告警阈值与机制： 定义何时、以何种方式（邮件、钉钉、短信）触发告警。
  • 迭代触发条件： 例如：“当线上AUC连续3天下降超过5%时，启动模型重训练流程。”
  • 数据回滚策略： 当新模型出现问题时，快速回滚到旧版本的方案。
  • A/B测试框架需求： 支持流量分割、多组实验、统计显著性检验的能力。
// 示例：模型监控告警配置（伪代码）
class ModelMonitorConfig:
    def __init__(self, model_id):
        self.model_id = model_id
        self.metrics = {
            'inference_latency_p99': {'threshold': 200, 'unit': 'ms'}, // P99延迟阈值200ms
            'daily_auc': {'threshold': 0.02, 'change': 'decrease'}, // AUC日环比下降超过2%
            'feature_drift_kl': {'threshold': 0.1, 'metric': 'KL_divergence'} // 特征分布KL散度>0.1
        }
        self.alert_channels = ['email', 'slack_critical_channel']
        self.check_frequency = 'hourly' // 检查频率

    def trigger_retraining(self):
        """触发重训练的条件判断"""
        if self.auc_drop_sustained() and self.data_drift_detected():
            return True
        return False

总结

模型需求与评估指标的定义是连接AI技术与业务价值的桥梁。产品专家需主导完成以下工作：

  1. 基于问题本质与数据现状，合理选择模型类型。
  2. 深入业务，定义可量化、可监控的核心评估指标，并确保模型指标与业务指标形成可验证的对齐关系。
  3. 为模型的全生命周期设计系统化的监控和迭代机制，将其作为产品需求不可或缺的一部分，保障AI系统持续、稳定地创造价值。

唯有如此，AI产品才能从一次性的项目，进化为可持续演进、不断适应变化的核心业务能力。
