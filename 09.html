第九讲：数据需求规格说明书

训练数据、推理数据与数据治理的核心要素

📊 数据需求规格说明书概述

数据是AI产品的“燃料”。清晰、完整的数据需求规格是项目成功的基石。本部分将详细阐述训练数据、推理数据以及数据管道与治理的核心需求。

核心目标：确保数据在正确的时间，以正确的格式、质量和规模，提供给正确的AI模型。

🔬 训练数据需求

训练数据用于“教导”模型，其质量直接决定模型性能的上限。

1. 数据来源

  • 公开数据集：如ImageNet、COCO、GLUE等，用于预训练或基准测试。
  • 业务系统生成：用户日志、交易记录、客服对话等内部数据。
  • 人工采集与合成：针对特定场景进行数据采集，或使用GAN、渲染技术合成数据。
  • 第三方购买/合作：在合规前提下，从数据供应商处获取。

2. 数据规模

模型类型建议最小数据量典型规模说明
简单分类模型1,000 - 10,000条万级每类别需足够样本
复杂视觉模型10,000 - 100,000张十万级需考虑图像分辨率和多样性
大语言模型(微调)10,000 - 1,000,000条对话/文档百万级高质量指令对至关重要
强化学习模型数百万步交互模拟环境生成依赖仿真环境效率

3. 数据质量要求

准确性

数据本身及其标注必须正确无误。错误样本比例应低于预定阈值（如<1%）。

一致性

相同语义的数据应有统一的格式和标注标准，避免矛盾。

完整性

关键字段无缺失，对于图像/语音数据，信号质量需达标（如无严重噪声）。

平衡性

分类任务中各类别样本量不宜差异过大，或需制定重采样策略。

4. 数据标注要求

  • 标注规范：需提供详细的《标注指南》，明确标签体系、边界案例处理方式。
  • 标注工具：选定或开发专用标注工具（如LabelImg, CVAT, Prodigy）。
  • 质量控制：设置多人标注、交叉验证、质检员抽检等流程。标注一致率（IoU/Kappa系数）需达到目标（如>0.9）。
  • 元数据：记录标注人员、时间、置信度等信息。

⚙️ 推理数据需求

推理数据是模型上线后处理的实际数据，关注输入输出的接口与性能。

1. 数据格式

// 示例：图像分类API请求格式 (JSON)
{
  "request_id": "uuid_123456",
  "timestamp": "2023-10-27T10:00:00Z",
  "data": {
    "image_base64": "/9j/4AAQSkZJRgABAQEAYABgAAD...",
    "model_version": "v2.1"
  }
}

// 示例：文本情感分析API请求格式
{
  "text": "这款产品的用户体验非常出色，响应速度很快。",
  "language": "zh-CN",
  "max_length": 512
}

常见格式：JSON, Protobuf, Base64编码的二进制文件（图像/音频），纯文本。

3. 数据预处理对齐

推理端的预处理（如归一化、分词、resize）必须与训练时完全一致。需提供标准化预处理脚本或容器。

🔄 数据管道与治理需求

构建可靠、可追溯、合规的数据流水线。

核心治理需求

  1. 版本控制：数据集、标注结果、模型代码需有唯一版本号，可追溯。
  2. 元数据管理：记录数据血缘（Lineage）、schema、统计信息、负责人。
  3. 质量监控：对输入推理数据分布进行监控，预警数据漂移（Data Drift）。
  4. 隐私与安全：遵守GDPR、网络安全法等，实施数据脱敏、加密存储、访问控制。
  5. 生命周期管理：制定冷热数据存储策略，定期归档与清理。

📝 总结与检查清单

  • ✅ 训练数据来源是否合法、多样且具代表性？
  • ✅ 数据规模是否匹配模型复杂度？是否制定了数据增强策略？
  • ✅ 标注规范是否详尽无歧义？质检流程是否明确？
  • ✅ 推理API的输入输出格式、性能SLA是否明确定义？
  • ✅ 数据管道是否具备版本化、可监控、可回溯的能力？
  • ✅ 隐私、安全与合规要求是否贯穿数据全生命周期？

一份优秀的数据需求规格说明书，是连接产品构想与技术实现的坚实桥梁。
