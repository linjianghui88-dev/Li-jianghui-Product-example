AI产品需求分析方法论

第八讲：AI功能与非功能需求定义

核心AI功能需求、性能需求、可解释性、公平性、安全性

一、核心AI功能需求定义

AI功能需求描述了系统“做什么”，特别是其核心的智能处理能力。这是AI产品区别于传统软件的核心。

1. 预测 (Prediction)

  • 定义: 基于历史数据和现有模式，对未来或未知状态进行估计。
  • 典型场景: 销量预测、股价趋势、用户流失预警。
  • 需求描述要点: 明确预测目标变量、时间范围、输入数据源、输出形式（概率值、具体数值、区间）。

2. 分类 (Classification)

  • 定义: 将输入数据划分到预定义的类别中。
  • 典型场景: 图像识别（猫/狗）、垃圾邮件过滤、情感分析（正面/负面）。
  • 需求描述要点: 定义所有类别标签、分类的粒度、处理多标签或分层分类的需求。

3. 生成 (Generation)

  • 定义: 创造新的、符合特定模式或要求的内容。
  • 典型场景: 文本创作（新闻、诗歌）、图像生成、代码补全、音乐合成。
  • 需求描述要点: 定义生成内容的格式、风格、长度、创造性程度约束、种子输入要求。

二、性能需求 (Performance Requirements)

衡量AI系统在特定条件下执行效果的量化指标，是评估模型优劣的关键。

指标类别 具体指标 定义与说明 典型基准要求
效果指标 准确率 (Accuracy) 模型预测正确的样本占总样本的比例。 > 95% (分类任务)
召回率 (Recall) 在所有正样本中，被模型正确识别出的比例。 > 90% (风控、医疗诊断)
效率指标 延迟 (Latency) 从输入提交到收到结果的时间间隔。 < 200ms (交互式应用)
吞吐量 (Throughput) 单位时间内系统能处理的请求数量。 > 1000 QPS (高并发场景)
资源指标 资源消耗 模型运行时的CPU、GPU、内存占用。 GPU内存 < 4GB (边缘设备)
注意：性能需求需根据具体业务场景权衡。例如，实时推荐系统对延迟要求极高，而离线数据分析则更关注准确率和召回率。

三、非功能需求：可解释性、公平性、安全性

这些需求关乎AI系统的可信度、社会责任和长期稳健性，是AI产品成功落地的关键保障。

1. 可解释性 (Explainability)

  • 目标: 让用户理解模型为何做出特定决策。
  • 需求描述:
    1. 全局解释: 提供模型整体决策逻辑的概述（如特征重要性排序）。
    2. 局部解释: 对单个预测结果，给出关键影响因素（如LIME、SHAP值）。
    3. 解释呈现: 通过可视化图表、自然语言描述等方式，向不同技术背景的用户提供解释。

2. 公平性 (Fairness)

  • 目标: 确保模型决策不因性别、种族、年龄等受保护属性而产生歧视性偏差。
  • 需求描述:
    1. 偏差检测: 定义需监控的敏感属性组，并设定公平性指标（如 demographic parity, equal opportunity）。
    2. 偏差缓解: 在数据预处理、模型训练或后处理阶段加入去偏技术。
    3. 持续审计: 建立模型上线后的长期公平性监控与报告机制。

3. 安全性 (Security)

  • 目标: 保护AI系统免受恶意攻击，确保其可靠运行。
  • 需求描述:
    1. 对抗攻击防御: 对输入数据进行鲁棒性检测，防御对抗样本攻击。
    2. 数据隐私: 采用联邦学习、差分隐私等技术保护训练数据隐私。
    3. 模型保护: 防止模型被窃取（模型提取攻击）或投毒（数据投毒攻击）。
    4. 内容安全: 对生成式AI的输出内容进行过滤，防止生成有害、违法信息。

四、需求定义示例（代码片段）

以下是一个AI文本分类需求定义的伪代码/配置文件示例，展示了如何结构化描述需求。

{
  “project”: “用户评论情感分析系统”,
  “core_functional_requirements”: {
    “type”: “classification”,
    “input”: {
      “format”: “raw_text”,
      “max_length”: 500,
      “language”: “zh-CN”
    },
    “output”: {
      “classes”: [“positive”, “neutral”, “negative”],
      “format”: “json”,
      “includes_confidence”: true
    }
  },
  “performance_requirements”: {
    “accuracy”: { “min”: 0.92, “target”: 0.95 },
    “recall_for_negative”: { “min”: 0.88 },
    “latency_p95”: { “max_ms”: 150 },
    “throughput”: { “min_qps”: 500 }
  },
  “non_functional_requirements”: {
    “explainability”: {
      “required”: true,
      “method”: “feature_importance”,
      “output_format”: “visual_chart”
    },
    “fairness”: {
      “sensitive_attributes”: [“none”], // 此场景不涉及
      “audit_frequency”: “quarterly”
    },
    “security”: {
      “input_sanitization”: true,
      “adversarial_detection”: “basic”
    }
  }
}
